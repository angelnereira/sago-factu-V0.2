# Reportes de Errores

Este directorio centraliza incidencias técnicas relevantes encontradas en el proyecto. Cada reporte documenta:

- **Contexto**: Cuándo y dónde apareció el problema.
- **Síntomas**: Mensajes, logs o comportamientos observados.
- **Causa raíz**: Diagnóstico confirmado.
- **Resolución**: Pasos aplicados para solucionarlo.
- **Verificación**: Cómo se validó que quedó resuelto.
- **Acciones preventivas**: Medidas para evitar recurrencia.

## Convenciones

- Los archivos siguen el formato `CASE-XXXX_descripcion-breve.md`.
- Usa Markdown y títulos de segundo nivel (`##`) para todas las secciones.
- Incluye comandos y evidencias relevantes (`logs`, `capturas`, enlaces).
- Actualiza el reporte si reaparece el mismo incidente con nueva información.
